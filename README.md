# vepub — 使用說明

vepub 是一款本機執行的 EPUB 有聲書閱讀器，整合 AI 語音合成（TTS）、場景插圖生成與角色管理功能。所有模型與資料均儲存於本機，不需連線。

## 環境需求

| 項目 | 需求 |
|------|------|
| OS | Windows 10/11（64-bit） |
| GPU | NVIDIA RTX 系列，VRAM ≥ 8 GB（建議 ≥ 16 GB） |

## 啟動開發模式

在專案根目錄執行：

```powershell
npm run dev
```

這會同時啟動兩個服務：

| 服務 | 位址 |
|------|------|
| FastAPI 後端 | http://127.0.0.1:8765 |
| Tauri + Vite 前端 | http://localhost:5173 |

**若遇到 port 被佔用**，執行以下指令清除殘留行程後再重試：

```powershell
Get-NetTCPConnection -LocalPort 5173,8765 -ErrorAction SilentlyContinue |
  ForEach-Object { Stop-Process -Id $_.OwningProcess -Force -ErrorAction SilentlyContinue }
```

**啟動畫面**：應用程式開啟後會顯示「後端啟動中…」遮罩，直到後端健康檢查通過（`GET /health` 回應 200）才進入書庫。最長等候 90 秒，逾時會顯示錯誤提示。

## 書庫

書庫是應用程式的首頁，管理所有匯入的電子書。

### 匯入書籍

- 點選右上角 **「匯入書籍」** 按鈕，選擇 `.epub` 檔案
- 或直接將 `.epub` 檔案**拖放**到書庫任意位置

解析完成後書籍封面、書名、作者會自動顯示。

### 搜尋

使用頂部搜尋欄，輸入書名或作者關鍵字即時篩選。

### 閱讀進度

每本書封面下方顯示閱讀百分比（依章節計算）與總章節數。

### 刪除書籍

滑鼠懸停於書籍封面時，右上角出現垃圾桶圖示，點選後確認即可刪除（書籍資料與檔案均一併移除，無法復原）。

### 其他入口

頂部導航列有兩個快捷按鈕：

- **模型管理**（綠色）→ 前往模型下載與設定頁面
- **插圖測試台**→ 獨立的插圖生成測試介面

## 閱讀器

點選任意書籍卡片進入閱讀器。

### 介面佈局

```
┌─────────────────────────────────────────────────────────┐
│ ← 返回  書名                        目錄  設定  角色管理  │  ← 頂部工具列
├─────────────────────────────────────────────────────────┤
│                                                         │
│   正文區（自動斷句、可點選任意句子跳播）                    │
│                                                         │
│   ● 插圖卡（若有生成圖片，出現在對應段落下方）             │
│                                                         │
├─────────────────────────────────────────────────────────┤
│          ⏮  ⏪  ▶/⏸  ⏩  ⏭   ══  音量  速度             │  ← PlayerBar（懸浮膠囊）
└─────────────────────────────────────────────────────────┘
```

### 章節切換

- 點選頂部 **目錄** 圖示展開側邊章節列表，點選任意章節跳轉
- PlayerBar 的 **⏮ / ⏭** 按鈕切換上/下章節

### 閱讀進度

自動儲存目前章節、句子位置與捲動位置，下次開啟同一本書自動恢復。

### 書籤

選取文字後出現浮動工具列，點選書籤圖示新增書籤；書籤可在設定面板查看。

### 文字選取工具列

選取任意文字後，浮動工具列提供：

| 按鈕 | 功能 |
|------|------|
| 生成插圖 | 以選取文字為場景描述，排隊生成一張插圖 |
| 提取角色 | 以選取文字推斷角色外貌特徵，帶入角色管理 |
| 新增書籤 | 在目前位置新增書籤 |

## 語音朗讀（TTS）

### 開始 / 暫停

- PlayerBar 中央 **▶ / ⏸** 按鈕控制播放
- 或直接**點選正文中任意句子**，從該句開始播放

### 播放模式

PlayerBar 支援三種 TTS 模式（在設定面板切換）：

| 模式 | 說明 |
|------|------|
| 標準 | 一次合成一句，低延遲 |
| 流式 | WebSocket 邊合成邊播放，更流暢 |
| 即時 | 最低延遲，適合快速預覽 |

### 速度 / 音量

PlayerBar 右側滑桿分別控制播放速度（0.5×–2.0×）與音量。

### 使用 TTS 的前提

需先在**模型管理**頁面下載並啟用至少一個 TTS 模型，否則播放時會提示「尚未載入模型」。

## 插圖生成

### 前提

需先在**模型管理**下載一個圖像模型（動畫風或寫實風）。

### 生成方式

**方式一：選取文字**
1. 在正文選取一段描述場景的文字
2. 浮動工具列點選「生成插圖」
3. 插圖排入佇列，生成完成後顯示於對應段落下方

**方式二：設定面板快捷生成**
1. 打開設定面板 → 插圖區塊
2. 輸入 Prompt 前綴後點選「生成」

### 查看 / 刪除插圖

- 插圖卡顯示於觸發段落下方，包含 Prompt 摘要與生成參數
- 滑鼠懸停插圖卡出現 **✕** 刪除按鈕

### 生成設定（設定面板 → 插圖）

| 設定 | 說明 |
|------|------|
| 風格前綴 | 附加於 Prompt 前的風格描述詞（8 種預設可選） |
| 寬 / 高 | 輸出解析度（預設 1024×1024） |
| 種子 | -1 表示隨機，固定值可重現相同構圖 |

## 角色管理

點選閱讀器右上角 **角色管理（人像圖示）** 開啟側邊角色面板。

### 自動分析全書角色

1. 面板頂部點選 **「分析全書」**
2. 後端以 LLM 逐章提取角色外貌特徵（性別、年齡、髮色、瞳色、體型、服飾…）
3. 進度條即時顯示；分析可中途中斷，下次繼續時會從斷點繼續（checkpoint 機制）
4. 完成後角色列表自動刷新

> 需先在模型管理下載 LLM（語言模型）才能使用分析功能。

### 手動新增 / 編輯角色

- 點選角色卡右上角 **編輯** 圖示開啟編輯 modal
- 所有欄位均可手動填寫或清空
- 點選「鎖定」防止自動分析覆寫

### 角色欄位一覽

| 分類 | 欄位 |
|------|------|
| 基本 | 性別、年齡段 |
| 臉部 | 膚色、臉型、髮色、髮型、瞳色、眼形 |
| 體型 | 體型、身高 cm、體重 kg、三圍、罩杯 |
| 服飾 | 時代風格、招牌服裝、色系、配件 |
| 特殊 | 特徵胎記、特殊能力、其他 |

### 多視角角色圖

1. 展開角色卡 → 點選 **「生成多視角」**
2. 選擇動畫 / 寫實風格，系統自動生成正面、側面、半身、全身四張圖
3. 生成中顯示進度，完成後圖片縮圖附於角色卡

### 管理工具

| 按鈕 | 功能 |
|------|------|
| 補完欄位 | 對空白欄位以 LLM 推算預設值 |
| 整理重複 | 偵測相同角色的別名，自動合併 |
| 批次刪除 | 勾選後一次刪除多個角色 |

## 模型管理

從書庫頂部導航點選「模型管理」（或路由 `/models`）進入。

### 模型類別

| 類別 | 用途 | 常見格式 |
|------|------|---------|
| 語音合成（TTS） | 朗讀書籍 | OmniVoice 系列 |
| 圖像生成（Image） | 插圖、角色圖 | SDXL `.safetensors`、Z-Image Turbo、Wan2.1 |
| 語言模型（LLM） | 角色分析、Prompt 擴寫 | `.gguf`（llama-server） |

### 新增模型

1. 切換至目標類別分頁
2. 點選 **「+ 新增模型」**
3. 貼上下載 URL（支援 HuggingFace、Civitai 等直鏈）
4. 點選「偵測」自動填入檔名與類別
5. 點選「開始下載」，進度條即時顯示下載速度與剩餘大小
6. 下載完成後點選 **「啟用」** 設為預設模型

### Civitai 下載

Civitai 部分模型需要 API Token：
1. 點選右上角 **「Key」** 圖示
2. 貼入您的 Civitai API Token 儲存

### 圖像模型載入 / 卸載

圖像模型佔用 VRAM，設定頁或模型管理頁可手動「載入」或「卸載」以釋放顯存。

### LLM 模型

LLM 以 llama-server 在背景執行，閒置 300 秒後自動關閉以釋放 VRAM。下次使用時自動重啟。需將 `llama-server.exe` 放置於 `backend/llama_bin/` 目錄。

## 正式打包（Sidecar 模式）

正式版本將 Python 後端打包為獨立可執行檔（PyInstaller onedir），由 Tauri 自動啟動與管理生命週期。

### 建置流程

**步驟一：打包 Python 後端**

```powershell
cd backend
uv run pyinstaller vepub-backend.spec
```

輸出目錄：`backend/dist/vepub-backend/`

**步驟二：手動放置 sidecar**

```powershell
# 將 PyInstaller 輸出複製到 src-tauri/resources/
Copy-Item -Recurse -Force backend\dist\vepub-backend `
  apps\desktop\src-tauri\resources\vepub-backend
```

**步驟三：建置 Tauri 桌面應用**

在 `tauri.conf.json` 加入 resources 條目後執行：

```powershell
npm run tauri build
```

> **注意**：`resources` 條目僅在正式打包時加入，開發模式（`npm run dev`）下保持 `"resources": []`，否則 Tauri build script 會因路徑不存在而報錯。

### 執行時行為

- 應用程式啟動時，Tauri（`lib.rs`）自動在背景啟動 `vepub-backend.exe`
- 若找不到 sidecar，會 fallback 嘗試 `.venv` 中的開發用 Python（僅開發環境有效）
- 關閉應用程式時，Windows Job Object 確保 Python 行程與其衍生的 `llama-server.exe` 子行程全部終止
- 後端 log 寫入 `~/.epub-tts/backend.log`

### 模型存放位置

| 環境 | models 目錄 | llama_bin 目錄 |
|------|------------|----------------|
| 開發 | `backend/models/` | `backend/llama_bin/` |
| 生產 | `~/.epub-tts/models/` | bundle 內的 `llama_bin/` |
